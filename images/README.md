# Architecture diagrams

Rendered from the mermaid diagrams in the "how the skills fit together"
reference (artifact built during development — see the root `README.md`'s
"How the pieces fit" section for the plain-text equivalent).

- `skill-architecture-overview.png` — the full page: plugin overview, legend,
  all four diagrams, and the reads/writes/depends-on summary table.
- `fig1-termbase-ecosystem.png` — every artifact `terminology-audit`,
  `sdd-terminology-audit`, and `vocabulary-contract` read or write, with the
  one real cross-skill dependency (`termbase.csv` → `vocabulary-contract`)
  distinguished from optional/human-gated paths.
- `fig2-docs-to-glossary.png` — the linear use case: audit developer docs
  once, render into agent context.
- `fig3-spec-driven-loop.png` — the cyclical use case: draft spec → ambiguity
  check → enforce on later drafts → round-trip after implementation → human
  approves candidates back into the termbase.
- `fig4-markdown-ia-sequence.png` — `markdown-ia`'s three skills, shown as
  independent reads of the same corpus rather than a dependency chain.

Generated once from static mermaid source with a local Playwright + mermaid
render — these are snapshots, not a build artifact regenerated by CI. If the
skills' inputs/outputs change, the diagrams should be redrawn to match.
